# CDN IP Checker

🌐 زبان‌ها: [English](README.md) | [فارسی](README_fa.md)

یک ابزار سبک، سریع و چندنخی با پایتون برای بررسی IPهای CDN است.

این ابزار می‌تواند IPها را در دو حالت بررسی کند:

- **Domainless / Empty SNI**: اتصال TLS بدون ارسال SNI
- **Domain Fronting**: اتصال TLS با SNI دلخواه و ارسال درخواست HTTP با همان Host

هدف ابزار این است که کمک کند سریع‌تر IPهایی را پیدا کنید که برای اتصال TLS، تانلینگ، یا ابزارهایی که از Empty SNI / Domain Fronting پشتیبانی می‌کنند قابل استفاده هستند.

---

## شروع سریع

ابتدا پروژه را کلون کنید یا فایل `cdn_ip_checker.py` را مستقیم دانلود کنید:

```bash
git clone https://github.com/FarazFe/cdn-ip-checker.git
cd cdn-ip-checker
```

یک فایل شامل IPها بسازید. هر خط باید فقط یک IP داشته باشد:

```bash
echo "23.209.22.214" > ips.txt
```

رایج‌ترین حالت استفاده، بررسی بدون SNI است:

```bash
python cdn_ip_checker.py -f ips.txt
```

آی‌پی‌های سالم در فایل زیر ذخیره می‌شوند:

```text
clean_domainless.txt
```

اگر می‌خواهید IPها را با یک SNI خاص هم تست کنید:

```bash
python cdn_ip_checker.py -f ips.txt --sni a248.e.akamai.net
```

برای تغییر تعداد تردها یا زمان timeout:

```bash
python cdn_ip_checker.py -f ips.txt -w 50 -t 3
```

## نحوه کار

اسکریپت برای هر IP دو مرحله اصلی را انجام می‌دهد:

1. **اتصال TCP به پورت 443**

   در این مرحله بررسی می‌شود که آیا IP از طریق پورت 443 در دسترس است یا نه.

2. **TLS Handshake**

   بسته به حالت اجرا، اتصال TLS به یکی از این دو روش انجام می‌شود:

   - **بدون SNI**

     در این حالت، درخواست TLS بدون فیلد SNI ارسال می‌شود. اگر سرور TLS handshake را کامل کند، آن IP در حالت domainless سالم در نظر گرفته می‌شود.

   - **با SNI**

     در این حالت، TLS handshake با SNI مشخص‌شده انجام می‌شود. سپس یک درخواست ساده HTTP `GET /` با `Host` مشابه همان SNI ارسال می‌شود. اگر سرور یک پاسخ معتبر HTTP برگرداند، آن IP در حالت fronting سالم در نظر گرفته می‌شود.

تست‌ها به صورت همزمان و با استفاده از thread pool انجام می‌شوند، بنابراین می‌توان تعداد زیادی IP را در زمان کوتاه بررسی کرد.

## ویژگی‌ها

- بررسی IPها در حالت **بدون SNI**
- پشتیبانی از تست **Domain Fronting** با SNI دلخواه
- اجرای چندنخی برای سرعت بیشتر
- بدون نیاز به کتابخانه‌های جانبی
- امکان تنظیم timeout و تعداد workerها
- ذخیره خودکار IPهای سالم در فایل‌های متنی
- مناسب برای تست و فیلتر کردن لیست‌های بزرگ IP

## نحوه استفاده

ابتدا یک فایل متنی آماده کنید که در هر خط آن یک IP قرار داشته باشد:

```text
23.209.22.214
104.16.1.1
151.101.1.69
```

خط‌هایی که با `#` شروع شوند به عنوان کامنت در نظر گرفته می‌شوند و نادیده گرفته می‌شوند:

```text
# Akamai test IPs
23.209.22.214
23.45.67.89
```

---

### اسکن بدون SNI

برای تست IPها بدون ارسال SNI:

```bash
python cdn_ip_checker.py -f ips.txt
```

در این حالت، هر IP بدون SNI تست می‌شود.

IPهای سالم در فایل زیر ذخیره می‌شوند:

```text
clean_domainless.txt
```

---

### اسکن با SNI

اگر می‌خواهید IPها را با یک SNI مشخص هم تست کنید، از گزینه `--sni` استفاده کنید:

```bash
python cdn_ip_checker.py -f ips.txt --sni a248.e.akamai.net
```

در این حالت، اسکریپت برای هر IP هر دو تست را انجام می‌دهد:

- تست بدون SNI
- تست با SNI مشخص‌شده

نتایج در دو فایل جدا ذخیره می‌شوند:

```text
clean_domainless.txt
clean_fronting.txt
```

---

### گزینه‌های پیشرفته

```text
-f, --file       مسیر فایل لیست IPها. اجباری است.
--sni            نام SNI برای تست fronting. اگر وارد نشود فقط تست بدون SNI انجام می‌شود.
-t, --timeout    زمان timeout برای هر تست، بر حسب ثانیه.
-w, --workers    تعداد تردهای همزمان برای تست IPها.
```

مثال با timeout کمتر و تعداد worker بیشتر:

```bash
python cdn_ip_checker.py -f akamai_ips.txt --sni example.com -t 3 -w 50
```

اگر شبکه کند یا ناپایدار است، می‌توانید timeout را بیشتر کنید:

```bash
python cdn_ip_checker.py -f ips.txt -t 10
```

```text
clean_domainless.txt
clean_fronting.txt
```

این فایل‌ها را می‌توانید مستقیماً در ابزارها یا کانفیگ‌های دیگر استفاده کنید.

## حالت Empty SNI چگونه کار می‌کند؟

در اتصال‌های معمول HTTPS، کلاینت هنگام TLS handshake معمولاً مقدار SNI را ارسال می‌کند. SNI به سرور می‌گوید کاربر قصد اتصال به کدام دامنه را دارد.

اما در حالت Empty SNI یا Domainless، کلاینت هیچ نام دامنه‌ای در TLS ClientHello ارسال نمی‌کند.

بعضی از سرورهای CDN همچنان این نوع اتصال را قبول می‌کنند و به جای رد کردن اتصال، handshake را کامل می‌کنند یا یک گواهی پیش‌فرض برمی‌گردانند.

در این ابزار، اعتبارسنجی گواهی TLS غیرفعال است، چون هدف بررسی معتبر بودن certificate نیست. هدف فقط این است که مشخص شود آیا IP می‌تواند TLS handshake را کامل کند یا نه.

به زبان ساده:

- در حالت Empty SNI، هیچ دامنه‌ای در handshake ارسال نمی‌شود.
- سرور ممکن است اتصال را قبول کند و TLS را کامل کند.
- بعد از کامل شدن TLS، برنامه می‌تواند پروتکل خودش را روی آن اتصال اجرا کند.
- این روش برای شرایطی مفید است که فیلترینگ یا محدودسازی بر اساس SNI انجام می‌شود.

## معنی نتیجه‌ها

اگر یک IP به عنوان `clean` نمایش داده شود، یعنی آن IP فقط در همان تست مشخص موفق بوده است.

### نتیجه سالم در حالت domainless

یعنی IP:

- روی پورت 443 قابل اتصال بوده است.
- TLS handshake را بدون SNI کامل کرده است.

### نتیجه سالم در حالت fronting

یعنی IP:

- روی پورت 443 قابل اتصال بوده است.
- TLS handshake را با SNI مشخص‌شده کامل کرده است.
- بعد از ارسال درخواست HTTP، یک پاسخ معتبر HTTP برگردانده است.

توجه کنید که `clean` بودن یک IP تضمین نمی‌کند که آن IP همیشه پایدار، سریع یا قابل استفاده در همه ابزارها باشد. رفتار CDNها ممکن است بسته به موقعیت جغرافیایی، مسیر شبکه، زمان، تنظیمات CDN و سیاست‌های سرویس‌دهنده تغییر کند.

## کاربردها

- فیلتر کردن لیست‌های بزرگ IP قبل از تست دستی
- پیدا کردن IPهایی که TLS بدون SNI را قبول می‌کنند
- تست IPهای CDN با SNI مشخص
- بررسی رفتار CDNها در شبکه‌ها و مسیرهای مختلف
- آماده‌سازی لیست IP برای ابزارهایی که از Empty SNI یا Domain Fronting پشتیبانی می‌کنند

## نکات استفاده مسئولانه

فقط IPها و سرویس‌هایی را تست کنید که اجازه بررسی آن‌ها را دارید.

اسکن تعداد زیاد IP می‌تواند باعث ایجاد ترافیک زیاد شود و ممکن است با قوانین یا شرایط استفاده برخی سرویس‌دهنده‌ها سازگار نباشد.

برای لیست‌های بزرگ، بهتر است از مقدارهای منطقی برای timeout و تعداد worker استفاده کنید.
